# ⛔ ПРАВИЛО №1 — ТЕСТ НА КАЖДОЕ ИЗМЕНЕНИЕ

## ОБЯЗАТЕЛЬНО после ЛЮБОЙ правки кода:

```
1. Написать тест (Python unittest или Vitest)
2. Убедиться что тест ПАДАЕТ без исправления (или описать почему это невозможно)
3. Убедиться что тест ПРОХОДИТ с исправлением
4. Запустить ВСЕ тесты (не только новые)
5. Только потом деплой
```

## Что покрывать тестами

| Тип изменения | Что тестировать |
|---|---|
| Новая функция / класс | Unit тест всех ветвей |
| Исправление бага | Тест воспроизводящий баг ДО фикса |
| Изменение поведения | Тест нового + тест что старое сломанное поведение исправлено |
| API / route | Тест endpoint (FastAPI TestClient) |
| Pipeline stage | Тест что stage завершается completed, а не failed |
| i18n ключ | Vitest что ключ присутствует и не пустой |
| Config / schema | Тест загрузки и default значений |

## Чек-лист перед деплоем

- [ ] Написаны новые тесты
- [ ] `PYTHONPATH=src python3 -m unittest discover -s tests -q` → OK
- [ ] `cd ui && npm run build && npm test` → passed
- [ ] Changelog заполнен
- [ ] `bump_version.py` вызван
