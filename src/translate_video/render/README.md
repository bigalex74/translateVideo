# Рендер

Сервисы рендера объединяют видео, исходное аудио, переведенную озвучку и
субтитры в итоговые артефакты. Первая реальная реализация может обернуть
устаревшее поведение MoviePy/ffmpeg, но пайплайн должен оставаться независимым
от провайдера.

`legacy.py` содержит MoviePy-рендерер закадровой озвучки.

## Политика Тайминга

Начиная с `TVIDEO-041` рендер не должен терять смысл перевода ради жёсткого
попадания в слот. Если TTS-аудио длиннее сегмента, рендер сначала применяет
pitch-safe ускорение через ffmpeg `atempo`. Если ускорение не помогает, аудио
по умолчанию сохраняется полностью, а сегмент получает QA-флаг
`render_audio_overflow`.

Обрезка через `subclip` разрешена только при явном
`allow_render_audio_trim=True`. Этот режим считается destructive-настройкой и
нужен только для специальных сценариев, где наложение аудио недопустимо.
