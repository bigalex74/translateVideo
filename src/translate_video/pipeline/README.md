# Пайплайн

Пакет пайплайна координирует провайдер-независимые этапы. Он записывает
запуски этапов и артефакты через `ProjectStore`, чтобы CLI, UI и будущие
webhook-интеграции могли единообразно продолжать и проверять задачи.

## Этап `timing_fit`

`timing_fit` выполняется после перевода и до TTS. Его задача — подготовить
переведённый текст к естественной озвучке без ускорения. Этап может сократить
очевидно избыточные обороты и записывает результат обратно в переведённый
transcript. Если безопасное сокращение невозможно, текст не режется, а сегмент
получает QA-флаг `timing_fit_failed`.

Этап пишет структурированные события `timing_fit.start`,
`timing_fit.segment_start`, `timing_fit.rewrite_attempt` и `timing_fit.done`.
Это нужно для диагностики длинных запусков: по логам видно, какой сегмент и
какой провайдер задерживает обработку.
