# Подгонка Таймингов

Пакет содержит логику подготовки перевода к естественной озвучке без ускорения
и обрезки аудио.

Главный принцип: сначала адаптировать текст под длительность сегмента, а уже
потом синтезировать речь. Если текст нельзя безопасно сократить, пайплайн
сохраняет полный смысл и ставит QA-флаг для проверки.

`natural.py` содержит основной `NaturalVoiceTimingFitter`.

`cloud.py` содержит fallback-роутер облачных моделей. Порядок по умолчанию:
Gemini → AIHubMix → OpenRouter → Polza.ai → rule_based. Polza.ai стоит
последним, потому что это платный агрегатор, и включается только при явном
`REWRITE_ALLOW_PAID_FALLBACK=true`. Если ключей нет или все провайдеры
недоступны, используется локальный `RuleBasedTimingRewriter`.

Промпт сокращения получает не только текущую фразу, но и 2 предыдущих/2
следующих сегмента, стиль перевода, уровень адаптации, аудиторию, предметную
область, глоссарий и список терминов, которые нельзя переводить.

Если `translation_quality=professional`, `timing_fit` не идёт по бесплатной
fallback-цепочке. Он использует только выбранные
`professional_rewrite_provider` и `professional_rewrite_model`.

Чтобы бесплатные лимиты не превращали этап в “вечное ожидание”, один сетевой
запрос ограничен `REWRITE_PROVIDER_TIMEOUT` секундами. После `429`, `503` или
timeout провайдер уходит во временный cooldown, а не отключается навсегда.
Для бесплатных моделей дополнительно действует консервативный RPM-лимит
`rewrite_provider_rpm`, чтобы не провоцировать новые `429`.

Реальные ключи задаются через окружение или локальный `.env`, который
игнорируется Git:

```bash
GEMINI_API_KEY=...
OPENROUTER_API_KEY=...
AIHUBMIX_API_KEY=...
POLZA_API_KEY=...
REWRITE_PROVIDER_TIMEOUT=8
REWRITE_ALLOW_PAID_FALLBACK=false
```
