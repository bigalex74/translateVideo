# TVIDEO-009: Локальный UI и Webhook API

Этот документ описывает архитектуру, технологический стек и план реализации для создания полноценного UI-интерфейса и API-слоя поверх ядра `translate_video`. Это позволит оркестрировать пайплайн как вручную через удобный интерфейс, так и автоматически (например, через n8n).

## 1. Технологический Стек

### Backend (API-слой)
Ядро уже написано на Python (3.11+), поэтому бэкенд будет логичным продолжением:
- **Фреймворк:** **FastAPI**. Идеально подходит для создания быстрых асинхронных API, имеет встроенную генерацию OpenAPI-документации (Swagger), что упростит интеграцию с n8n.
- **Сервер:** **Uvicorn** (ASGI-сервер для FastAPI).
- **Хранение данных:** Мы продолжим использовать файловую систему (`ProjectStore` и `project.json`) как единый источник истины. FastAPI будет выступать как прослойка над `runner.py` и `store.py`.
- **Фоновые задачи:** `asyncio.create_task` или легковесный `BackgroundTasks` из FastAPI для запуска пайплайна без блокировки HTTP-ответов. Для более сложных сценариев в будущем можно рассмотреть Celery/Redis, но для локального MVP хватит встроенных средств.

### Frontend (Пользовательский интерфейс)
- **Фреймворк:** **React** (через **Vite** для быстрой сборки). Это обеспечит высокую реактивность интерфейса.
- **Язык:** **TypeScript** для строгой типизации контрактов, совпадающих со схемами Python (Project, Segment, Status).
- **Стилизация:** **Vanilla CSS** (или CSS Modules). Это соответствует внутренним правилам минимизации внешних зависимостей (Tailwind не используется без явной необходимости).
- **Управление состоянием:** `Zustand` (или просто `React Context`, учитывая локальность проекта).
- **Связь с API:** `fetch` или `Axios`. Использование Server-Sent Events (SSE) или WebSockets для потоковой передачи логов пайплайна и обновления статуса (`PENDING` -> `RUNNING` -> `COMPLETED`) в реальном времени.

### CI/CD
- **Платформа:** **GitHub Actions** (или локальные скрипты, если GHA недоступен).
- **Процесс (Pipeline):**
  - Идентично текущему бэкенду: строгие уровни тестирования (Модульные -> Интеграционные -> E2E).
  - *Lint & Format:* Проверка Python-кода (Ruff, Black) и TypeScript-кода (ESLint, Prettier).
  - *Test:* Прогон модульных и интеграционных тестов Python (`unittest`, `coverage`), а также базовых тестов React (`Vitest`).
  - *Build:* Сборка frontend-а через Vite.
  - *Docker:* Сборка единого Docker-образа при создании релизного тега.

### Deployment (Деплой)
- **Standalone (Локальное использование):** Frontend компилируется в статику (папка `dist`), а FastAPI раздает эти файлы через `StaticFiles`.
- **Docker Compose & Public Access:** Приложение будет упаковано в `docker-compose.yml` (со всеми зависимостями вроде FFmpeg) и развернуто на сервере.
  - **Домен:** Приложение будет доступно из интернета по адресу `https://video.bigalexn8n.ru`.
  - **Reverse Proxy:** Существующий инстанс Caddy на хосте `bigalexn8n.ru` будет настроен на проксирование трафика к контейнеру с приложением (порт 8000), автоматически обеспечивая HTTPS.

## 2. Дизайн и UX (User Experience)

### Общая концепция
- **Стиль:** Минималистичный, темная тема (Dark Mode) по умолчанию — привычно для создателей контента и видеоредакторов.
- **Навигация:** Боковая панель для переключения между «Новый проект», «История проектов», «Настройки».

### Основные экраны
1. **Dashboard (История проектов):**
   - Таблица/карточки ранее запущенных проектов.
   - Столбцы: ID проекта, статус (с цветовой индикацией: зеленый - готов, желтый - в процессе, красный - ошибка), исходный язык, целевой язык, дата.
   - Вызов всех функций приложения: инициализация, предпроверка (preflight), статус, список артефактов и конфигурация.

2. **Создание проекта (Wizard):**
   - Шаг 1: Загрузка видео (Drag & Drop) или вставка URL.
   - Шаг 2: Настройка конфигурации (Source/Target язык, режим перевода, стиль, адаптация, стратегия голосов). Поддержка всех опций ядра.
   - Кнопка "Запустить перевод".

3. **Workspace (Рабочая область проекта):**
   - **Слева:** Полноценный видеоплеер. Возможность мгновенно оценить работу. Переключатель дорожек: оригинальное видео, видео с наложенной озвучкой, видео с субтитрами.
   - **Центр:** Интерактивный список сегментов (субтитров).
     - Визуализация таймкодов, исходного текста и перевода.
     - *Ключевая фича:* Возможность вручную отредактировать `translated_text` (полезно для этапа "Возобновление проекта").
   - **Справа:** Прогресс пайплайна. Вертикальный таймлайн (Extract Audio -> Transcribe -> Translate -> TTS -> Render), показывающий текущий статус каждого `StageRun`. Зона для вывода логов ошибок.

## 3. Архитектура Webhook / API
Для интеграции с n8n (как планировалось) API должен поддерживать асинхронную работу:
1. `POST /api/v1/projects` — Создать проект (отправить настройки и ссылку на видео).
2. `POST /api/v1/projects/{id}/run` — Запустить пайплайн. Если передан заголовок `X-Webhook-Url`, FastAPI отправит POST-запрос на этот URL по завершении (или при падении) проекта.
3. `GET /api/v1/projects/{id}` — Получить статус.
4. `GET /api/v1/projects/{id}/artifacts/{kind}` — Скачать готовое видео или SRT.
5. Экспонирование всех CLI-аналогов: `/preflight`, `/artifacts`, `/config`, `/export-srt`, `/timing-report` и т.д.
