# Дорожная Карта

## Завершённые Этапы

### 0.1.0 Модель Ядра

- Структура пакета.
- Схемы проекта и сегментов.
- Хранилище артефактов.
- Конфигурация пайплайна.
- Сфокусированные модульные тесты.

### 0.2.0 Сервисы Пайплайна Ядра

- Интерфейсы провайдеров для медиа, речи, перевода, TTS и рендера.
- Последовательный раннер пайплайна.
- Интеграционный путь с имитационными провайдерами.
- Запись запусков этапов и артефактов.

### 0.3.0 CLI

- Точка входа `translate-video`.
- Запуск текущего пайплайна через новое ядро.
- Рабочие папки для каждого видео.
- Команды просмотра статуса, артефактов и настроек.

### 0.4.0 Адаптеры Устаревшего Скрипта

- Реальные адаптеры медиа, распознавания речи, перевода, TTS и рендера.
- Подключение поведения `main.py` к контрактам ядра.
- Выбор набора провайдеров в CLI.

### 0.5.0 Предварительная Проверка Реального Видео

- CLI-команда `preflight`.
- Проверка входного файла без запуска перевода.
- Проверка legacy Python-зависимостей.
- Проверка FFmpeg/FFprobe.

### 0.6.0 Редактируемые Артефакты

- Импорт и экспорт JSON.
- Экспорт SRT/VTT.
- Артефакт проверки перевода.
- Отчет по таймингам.

### 0.7.0 Локальный UI

- Базовый React/Vite UI.
- Загрузка видео и создание проекта.
- Дашборд с ручной загрузкой проекта по ID.
- Рабочая область с видеоплеером, сегментами и прогрессом пайплайна.

### 0.8.0 API-Контракт И Артефакты

- Список проектов через `GET /api/v1/projects`.
- Полный payload проекта для UI.
- Сохранение отредактированных сегментов через API.
- Скачивание артефактов по типу.
- Предварительная проверка входного файла через API.

### 1.0.0 Автоматическая QA-Система

- Детерминированные проверки итогового медиа.
- Лингвистическая QA-оценка перевода.
- Audio QA-оценка громкости, тишины и таймингов.
- Единый `qa_report.json` как релизный артефакт проекта.

### 1.1.0-1.5.1 Стабилизация UI/API/Render

- Реальный provider `legacy` стал дефолтом вместо `fake`.
- Исправлены пути артефактов, видео route, MIME/Range отдача.
- Добавлен поллинг статуса и статус-зависимые кнопки запуска.
- Исправлен resume пайплайна.
- Улучшены bulk translate, pitch-safe speed и pre-trim TTS.
- Release gate восстановлен в `1.5.1`.

### 1.6.0 Локализация, QA Metadata И Fullstack E2E

- Русский UI по умолчанию, переключение интерфейса на английский.
- Машинно-читаемые QA-флаги деградаций на сегментах.
- QA-сводка отображает технические предупреждения.
- Browser E2E на моках и fullstack browser E2E против реального FastAPI backend.
- `make test:release` стал единым релизным gate.

## Следующие Этапы

### 1.7.0 Автономная AI-QA Проверка

- Агент проверки качества должен анализировать `qa_flags`, `qa_report.json`,
  тайминги, артефакты и статус этапов.
- Результат проверки должен быть машинно-читаемым и видимым в UI.
- Цель: система сама сообщает, можно ли публиковать результат.

### 1.8.0 Расширение Реальных Провайдеров

- Явные провайдеры ASR/translation/TTS/render в конфигурации.
- Проверка доступности моделей и сервисов до запуска.
- Возможность выбирать качество/скорость/стоимость обработки.

### 1.9.0 Оркестрация И Webhook-Интеграции

- Расширение webhook-событий.
- Подготовка внешнего orchestration API для n8n или другого менеджера задач.
- Идемпотентность событий и повторная доставка.

### 2.0.0 Продуктовый Релиз

- Стабильный публичный API.
- Документированный формат артефактов.
- Миграции проекта при изменении схем.
- Полный пользовательский сценарий без ручного участия в переводе.
