## Профиль 1 — Саша, 17 лет, блогер (Shorts/TikTok/YouTube)

> Снимает на телефон, монтирует в CapCut, хочет перевести ролик с русского на английский чтобы зайти в аудиторию США. Не знает что такое «провайдер».

**Z1.1** — Wizard создания проекта — огромный шаг вперёд (пресеты «Shorts/TikTok», «Лекция» и т.д.), но пресет «Shorts/TikTok» не даёт никаких объяснений что именно будет сделано. Нет превью-описания: «ролик будет переведён и озвучен за ~2 минуты». Саша нажимает наугад.

**Z1.2** — Загрузить видео можно файлом или URL, но нет минимальных требований: максимальный размер? поддерживаемые форматы? что будет с вертикальным 9:16? Страх «а вдруг не то загружу».

**Z1.3** — После запуска — оверлей «Выполняется перевод…» с названием этапа. Хорошо! Но нет ETA («ещё ~2 мин»). Саша уходит в другую вкладку и не знает, ждать 2 минуты или 20.

**Z1.4** — Workspace открывается в список сегментов. Вертикальное видео (9:16) показывается как горизонтальное в плеере — обрезается. Сразу негативное впечатление от качества результата.

**Z1.5** — Кнопка «▶ Прослушать» есть у каждого сегмента — отлично. Но скорость воспроизведения фиксирована, нельзя оценить темп озвучки целиком за 10 секунд.

**Z1.6** — «Скачать» не выделена как главная кнопка. Артефакты прячутся на вкладке «Артефакты» в правом сайдбаре. Саша ищет «где файл» несколько минут.

**Z1.7** — Режим «Только субтитры» есть, но нет режима «Субтитры + оригинальная озвучка» (для karaoke-стиля, популярного в TikTok с двуязычным звуком).

**Z1.8** — Нет прямой кнопки «Поделиться ссылкой на скачивание» для Telegram. Кнопка 📋 «Скопировать API URL» — маленькая и незаметная.

**Z1.9** — Нет советов по публикации: «YouTube требует субтитры в .vtt, TikTok — встроенные». Саша не знает какой артефакт выбрать.

**Z1.10** — В мобильном браузере интерфейс ломается: сайдбар скрывается, сегменты не редактируются. Критично для аудитории Shorts, которая работает с телефона.

## Профиль 2 — Максим, 34 года, фрилансер-видеомонтажер

> Работает в DaVinci Resolve, клиенты платят за ролик. Хочет перевести клиентское промо с английского на русский, сдать точно в срок, без сюрпризов.

**Z2.1** — Нет оценки стоимости до запуска. ConfirmRunModal показывает выбор этапа, но не говорит «это обойдётся ~$0.40 через DeepSeek + Polza». Максим боится выставить счёт клиенту неточно.

**Z2.2** — Выбор провайдера глобален, а не по проекту. Если работаешь с двумя клиентами параллельно на разных провайдерах — нельзя. Всё общее.

**Z2.3** — Нет журнала действий по проекту: кто что делал, когда запускал, какие этапы перезапускал. Для клиентской работы нужна трассировка.

**Z2.4** — Редактор сегментов не поддерживает Ctrl+Z (undo). Случайно перетёрли правку — нет истории. Критичный риск при длинных интервью (1+ час, 200+ сегментов).

**Z2.5** — Вкладка «QA» показывает флаги (`tts_voice_nova`, `timing_fit_failed`) — это технический язык. Максим хочет: «сегмент 12 — озвучка выходит за кадр на 0.8 сек, исправьте».

**Z2.6** — Нет экспорта отдельной аудиодорожки (WAV/MP3) для сведения в DaVinci Resolve. Артефакты: только видео + SRT. Нельзя взять только голос.

**Z2.7** — Время этапов не показывается наглядно. `stage_runs` есть в API, вкладка Stats неочевидна. Сколько реально заняло TTS — непонятно для контроля расписания.

**Z2.8** — Нет защиты от «force re-run» всего пайплайна: можно перезаписать уже одобренный перевод. ConfirmRunModal не предупреждает «существующий перевод будет удалён».

**Z2.9** — Глоссарий/список «не переводить» не реализован в UI. `do_not_translate` есть в конфиге, но в форме нет поля. Для клиентских видео с брендами («Кока-Кола», «iPhone») критично.

**Z2.10** — Нет поля «Клиент / Заказ» для организации работы. Дашборд не фильтруется по клиенту — при 20+ проектах найти нужный сложно.

## Профиль 3 — Ирина, 48 лет, владелец онлайн-школы

> Записывает вебинары по 2 часа, хочет выложить курс на Udemy и в зарубежных соцсетях. Работает с командой.

**Z3.1** — Нет поддержки длинных видео в UI: при загрузке 2-часового вебинара нет прогресс-бара закачки и нет предупреждения «обработка займёт 40-60 минут». Ирина думает что зависло.

**Z3.2** — Нет совместного доступа к проекту. Ирина не может отправить ассистенту ссылку «вот воркспейс, поправь перевод». Приложение без авторизации и без шеринга.

**Z3.3** — Пресет «Вебинар / Презентация» не предлагает опцию «сохранить главы по слайдам». Длинные вебинары теряют структуру при переводе.

**Z3.4** — Нет интеграции с платформами: «Загрузить субтитры на YouTube», «Экспорт для Udemy». Ирина копирует файл вручную и не знает правильный формат для каждой платформы.

**Z3.5** — Стоимость перевода 2-часового видео нигде не отображается до запуска. После завершения тоже нет чека. Ирина не знает сколько потратила и не может учесть в расходах.

**Z3.6** — Нет уведомлений (email/Telegram) о завершении. 2 часа — нужно переключиться на другие задачи. Webhook есть, но требует настройки n8n, которую Ирина не умеет.

**Z3.7** — Слайдер «Скорость речи: 1.30× (~14 симв/с)» бессмысленен без контекста. Нужно словесное описание: «нормальная» / «чуть быстрее» / «быстрая».

**Z3.8** — Нет шаблонов настроек. Каждый раз нужно заново выбирать язык, голос, провайдер TTS. Для серийного производства 10+ вебинаров в месяц — критично.

**Z3.9** — Нет согласия об обработке данных перед запуском. Ирина не понимает, куда уходит запись вебинара с участниками, кто её хранит — юридический риск.

**Z3.10** — После завершения нет понятного «итогового резюме»: потрачено X минут, качество Y%, проблемные сегменты N штук. Нужно самостоятельно разбирать QA-вкладку.

## Профиль 4 — Николай Иванович, 67 лет, преподаватель

> Записал лекцию для иностранных студентов. Компьютер использует только для email и Word. Низкая цифровая уверенность.

**Z4.1** — Переключатель «Загрузить файл / Указать путь к файлу» сразу ставит в тупик. «Путь к файлу» — что это? Нужна только одна большая кнопка «Выбрать видео с компьютера».

**Z4.2** — Шаг 2 Wizard «Настройки перевода» содержит 10+ параметров: «Дубляж / Субтитры / Дубляж + субтитры» без объяснений разницы. Николай Иванович зависает.

**Z4.3** — Тёмная тема по умолчанию. Серый текст на тёмном фоне — проблема читаемости для пожилых. Светлая тема есть в настройках, но найти настройки — отдельная задача.

**Z4.4** — Ошибки отображаются техническим текстом («tts.openai.error: timeout»). Николай Иванович не знает что делать. Нужно: «Не удалось создать озвучку. Попробуйте снова.»

**Z4.5** — Кнопка «Крупный текст» есть в настройках, но пользователь с плохим зрением не найдёт её без помощи — нет онбординга и нет подсказки на главном экране.

**Z4.6** — Нет обучающего тура при первом входе. Пустой дашборд — полная тишина, только маленькая кнопка «+ Новый проект» в шапке.

**Z4.7** — Уведомление «✅ Сохранено» мелкое и быстро пропадает. Николай Иванович не уверен что правки сохранились.

**Z4.8** — Кнопки «Запустить перевод» и «Перезапустить» выглядят одинаково. Нажимает «Перезапустить» думая что «Продолжить» — теряет уже сделанные правки.

**Z4.9** — Иконки без подписей (стрелка назад, шестерёнка, скачивание). Для пожилых пользователей нужны текстовые подписи под ключевыми иконками.

**Z4.10** — Нет кнопки «Помощь» или «Инструкция». Нет FAQ, нет контекстных подсказок «?». Единственный способ понять приложение — методом проб и ошибок.

## Профиль 5 — Дмитрий, 29 лет, DevOps / автоматизатор

> Настраивает n8n, хочет пачечно переводить подкасты, дёргать API, получать webhook, мониторить через Prometheus.

**Z5.1** — API документации нет. Нет `/docs` (Swagger/OpenAPI) в открытом доступе. Чтобы понять эндпоинты — нужно читать исходники FastAPI. Для B2B неприемлемо.

**Z5.2** — Нет аутентификации API. Все эндпоинты открыты без токена. На продакшн-сервере это дыра безопасности — нет ни Basic Auth, ни Bearer token, ни API key middleware.

**Z5.3** — Webhook глобален и без event types. Нет per-project webhook, нет фильтрации по событиям (`on_complete`, `on_error`, `on_stage_done`). n8n получает один безструктурный сигнал.

**Z5.4** — Нет batch API. Нельзя передать список файлов и получить очередь. Для пакетной обработки 50 подкастов — ручной труд: 50 отдельных POST `/projects`.

**Z5.5** — Нет progress_percent и ETA в polling: `GET /projects/:id` возвращает `status=running` без `progress_percent` и `estimated_completion`. Невозможно построить нормальный мониторинг в n8n.

**Z5.6** — Нет `/metrics` Prometheus. Нет gauge для активных задач, counter ошибок по провайдерам, histogram latency по этапам. Grafana не подключить без кастомного экспортёра.

**Z5.7** — Нет `trace_id` / `execution_id` в ответах API. Нельзя скоррелировать запрос клиента с логом сервера — несмотря на структурированный JSON в логах контейнера.

**Z5.8** — Нет `GET /projects/:id/stage-runs` с деталями (duration, cost, input_tokens per stage). Невозможно построить финансовую отчётность по проектам.

**Z5.9** — Нет retry API: если TTS упал — нужно вручную открывать UI. Нет `POST /projects/:id/retry?from_stage=tts`. Автоматический retry в n8n невозможен.

**Z5.10** — Docker-контейнер не экспортирует VERSION как label. `docker inspect video-translator` не показывает версию приложения — сложно управлять rollback и canary-деплоями в CI/CD.

### Новые Приоритеты — Backlog

#### P0: Критично До Реального Использования

- **ETA в прогрессе** — `estimated_completion` в API + отображение в оверлее
- **Стоимость до запуска** — `cost_estimate` в preflight/ConfirmRunModal + итоговый чек
- **Undo/история правок** — local undo (Ctrl+Z) + версии сегментов в JSON
- **Понятный QA-итог** — «3 сегмента выходят за тайминг» вместо флагов
- **Безопасность API** — `X-API-Key` middleware для продакшн-инсталляций

#### P1: Нужно Для Beta

- **progress_percent + ETA в `GET /projects/:id`** — для n8n polling
- **Шаблоны настроек** — «Сохранить конфиг как шаблон»
- **Онбординг** — подсказки при первом входе, empty state с CTA
- **Мобильный layout** — адаптация редактора и плеера под 9:16
- **Экспорт аудиодорожки** (MP3/WAV) отдельно от видео
- **Слова вместо чисел** в слайдерах скорости TTS («нормальная» / «быстрее»)

#### P2: Нужно Для Масштабирования

- **Swagger `/docs`** — автогенерация FastAPI (1 строка кода)
- **Batch API** — `POST /projects/batch` со списком файлов
- **Prometheus `/metrics`** — gauge задач, counter ошибок, histogram latency
- **Per-project webhook** с event types
- **Глоссарий в UI** — поле `do_not_translate` в форме
- **Совместный доступ** — шеринг проекта по ссылке
