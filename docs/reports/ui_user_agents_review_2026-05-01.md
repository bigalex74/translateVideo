# Отчет Пользовательских Агентов По UI

Дата: 2026-05-01

Этап: `TVIDEO-012-api-contract-and-artifacts`

Цель: критически оценить текущий UI приложения AI Video Translator с точки
зрения разных пользовательских профилей и выделить продуктовые риски.

## Профили Агентов

- 17 лет, начинающий блогер, переводит Shorts/TikTok/YouTube.
- 34 года, фрилансер-видеомонтажер, работает с клиентскими роликами.
- 48 лет, владелец онлайн-школы, переводит вебинары и обучающие материалы.
- 67 лет, преподаватель/пенсионер, низкая цифровая уверенность.
- 29 лет, DevOps/автоматизатор, хочет подключить n8n/API/batch.

## Общий Вывод

Текущий UI уже решает базовый технический сценарий: создать проект, увидеть
статус, открыть workspace, поправить сегменты и скачать доступные артефакты.
Но для реальных пользователей интерфейс пока выглядит как инженерная панель, а
не как продукт для безопасного перевода видео. Главные риски: непонятность
стоимости и времени, технический язык, отсутствие сценариев под тип пользователя,
слабая защита от дорогих ошибок и недостаточный контроль качества результата.

## Повторяющиеся Замечания

### 1. Слишком Технический Язык

Повторялось у всех нетехнических профилей. Слова `Дашборд`, `Воркспейс`,
`пайплайн`, `артефакты`, `QA-отчет`, `ID проекта`, `transcribe`, `tts`, `render`
создают ощущение инструмента для разработчика.

Решение:

- Переименовать основные разделы: `Дашборд` -> `Мои переводы`, `Воркспейс` ->
  `Редактирование перевода`.
- Добавить человекочитаемые названия этапов: `transcribe` -> `Распознавание
  речи`, `translate` -> `Перевод`, `tts` -> `Создание озвучки`.
- Скрыть технический `project_id` за пользовательским `Название проекта`, а ID
  показывать в расширенных параметрах.

### 2. Нет Прогноза Стоимости, Времени И Рисков До Запуска

Профили блогера, монтажера и владельца школы отдельно указали страх случайно
запустить дорогую обработку. Сейчас кнопки запуска и перезапуска не объясняют,
что будет пересчитано, сколько это займет и какие внешние сервисы будут
использованы.

Решение:

- Добавить `cost_estimate` и `duration_estimate` в API preflight.
- Показывать перед запуском длительность видео, примерную стоимость, ETA,
  выбранных провайдеров и предупреждение о повторной оплате при `force`.
- Для перезапуска добавить подтверждение и выбор этапов: перевод, TTS, рендер.

### 3. Недостаточно Пошаговости В Создании Проекта

Новые пользователи хотят мастер: загрузка -> языки -> результат -> проверка ->
запуск. Текущая форма показывает все сразу и не объясняет последствия выбора.

Решение:

- Переделать `NewProject` в wizard.
- Добавить пресеты: `Shorts/TikTok`, `Лекция`, `Интервью`, `Вебинар`,
  `Клиентский ролик`, `Только субтитры`.
- Для каждого режима показывать простое объяснение и рекомендуемый сценарий.

### 4. Слабые Настройки Реального Перевода

В ядре уже есть `translation_style`, `adaptation_level`, `voice_strategy`,
`terminology_domain`, `target_audience`, `do_not_translate`, но UI показывает
только язык и режим. Для бизнеса, монтажера и блогера этого недостаточно.

Решение:

- Вынести расширенную конфигурацию в UI.
- Добавить глоссарий, список `не переводить`, стиль, аудиторию, степень
  адаптации, правила обсценной лексики.
- Добавить выбор голоса, количества спикеров, предпрослушивание и пресеты
  голосов.

### 5. Нет Доверия К Качеству Итогового Видео

Пользователи хотят понять, можно ли публиковать результат. Сейчас есть ссылки на
артефакты, но нет понятного verdict: готово, есть риски, нужна ручная проверка.

Решение:

- Сделать `QA Summary` в workspace: качество текста, тайминги, голос,
  громкость, пустые сегменты, низкая уверенность ASR.
- Добавить экран приемки: оригинал и перевод рядом, список проблем, кнопки
  `Принять`, `Перегенерировать`, `Исправить перевод`.
- В дашборде показывать финальный статус качества.

### 6. Редактор Сегментов Нуждается В Безопасности Правок

Монтажер и пожилой преподаватель боятся потерять или испортить правки. Сейчас
есть сохранение, но нет undo, истории, сравнения версий и статуса по сегментам.

Решение:

- Добавить локальный undo/redo.
- Показывать diff: исходный ИИ-перевод -> правка пользователя.
- Вести версии `transcript.translated.json`.
- Добавить `Вернуть исходный перевод` и `Сохранено` по каждому сегменту.

### 7. Workspace Не Достаточно Удобен Для Монтажа И Мобильных Видео

Фрилансер хочет клики по таймкодам, подсветку активного сегмента, горячие
клавиши. Блогер хочет вертикальный режим и быструю кнопку скачивания.

Решение:

- Синхронизировать видеоплеер с сегментами.
- Клик по таймкоду должен перематывать видео.
- Добавить hotkeys: следующий сегмент, сохранить, replay текущего сегмента.
- Добавить адаптивный мобильный layout и режим вертикального предпросмотра.

### 8. Экспорт И Артефакты Пока Слишком Простые

Ссылки скачивания есть, но пользователи хотят управляемые deliverables: форматы,
версии, размер, checksum, дата, копирование API URL.

Решение:

- Добавить карточки артефактов с размером, checksum, датой, форматом.
- Добавить экспорт SRT/VTT/ASS, MP4 с озвучкой, отдельная аудиодорожка,
  stems, JSON.
- Добавить кнопку `Скопировать ссылку для n8n/API`.

### 9. Интеграции И Автоматизация Пока Недостаточны

DevOps-профиль указал, что webhook в настройках декоративный, provider
захардкожен как `fake`, нет batch, токенов, очереди и observability.

Решение:

- Сделать persisted settings для webhook URL и provider profile.
- Передавать webhook URL при запуске пайплайна.
- Добавить API tokens, scopes и audit log.
- Добавить batch queue: список файлов, параллельность, retry, pause/resume.
- Добавить diagnostics tab: логи, trace id, длительность этапов, версии моделей.

### 10. Недостаточно Юридических И Приватностных Предупреждений

Блогер и владелец школы отдельно отметили авторские права и приватность. UI не
говорит, куда уходит видео, какие внешние сервисы используются, можно ли удалить
данные и какие риски публикации.

Решение:

- Добавить блок согласия перед запуском: права на видео, голос, публикацию.
- Показывать список внешних провайдеров и типы данных, которые им отправляются.
- Добавить удаление проекта и артефактов.
- Добавить режим `личное использование` и предупреждения для YouTube/TikTok.

## Приоритетный Backlog

### P0: Нужно До Реального Использования

- Человеческие названия экранов, этапов и ошибок.
- Подтверждение запуска/перезапуска с прогнозом стоимости и времени.
- Расширенная форма настроек перевода: стиль, адаптация, голос, глоссарий.
- QA Summary в workspace с понятным verdict.
- Безопасное сохранение правок: undo и версии сегментов.

### P1: Нужно Для Beta

- Wizard создания проекта и пресеты сценариев.
- Синхронизация сегментов с видеоплеером.
- Карточки артефактов с форматами, размером, checksum и API URL.
- Настройки webhook/provider, которые реально сохраняются и используются.
- Светлая тема, крупный шрифт и режим повышенного контраста.

### P2: Нужно Для Масштабирования

- Batch queue и пакетная загрузка.
- API tokens, scopes, audit log.
- Observability: trace id, логи, duration по этапам, retry.
- Mobile/vertical preview для Shorts/TikTok.
- Экран бизнес-отчетности: стоимость, длительность, клиент, курс, статус QA.

## Рекомендация По Следующим Этапам

Следующий этап после TVIDEO-012 стоит делать не как новую модель перевода, а как
`TVIDEO-013-ux-language-and-safe-run`: привести язык UI к пользовательскому,
добавить wizard, подтверждение запуска, человекочитаемые статусы, базовый
прогноз стоимости/времени и безопасный перезапуск. Это снизит главный барьер
для всех профилей сразу.

После этого логично делать `TVIDEO-014-quality-review-workspace`: QA Summary,
синхронизация сегментов с видео, undo/versions и улучшенный экспорт.
