# Ручное Тестирование

Этот документ описывает проверки, которые можно запускать вручную на реальных
медиафайлах. Такие файлы не добавляются в git и не должны попадать в
автоматические unit/integration-gates.

## Проверка Реального Видео Без Перевода

Перед тяжелым запуском пайплайна используйте `preflight`. Команда проверяет
существование входного файла, Python-зависимости выбранного провайдера и
системные команды FFmpeg/FFprobe.

Пример для локального тестового файла:

```bash
PYTHONPATH=src python3 -m translate_video.cli preflight \
  "/home/user/Загрузки/Google_Antigravity__The_End_of_Coding_(And_The_Rise_of_the_IT_B.mp4" \
  --provider legacy
```

Пока отчет содержит `"ok": false`, реальный перевод запускать не нужно:
сначала нужно установить недостающие зависимости или подключить нужное
окружение.

## Правило Для Больших Файлов

- Не коммитить `.mp4`, `.wav`, `.mp3` и другие тяжелые медиафайлы.
- Использовать реальные видео только в ручных smoke-проверках или будущих
  opt-in E2E-тестах.
- Для автоматических тестов оставлять имитационные провайдеры и маленькие
  сгенерированные фикстуры.
